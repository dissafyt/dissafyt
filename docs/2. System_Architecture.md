# Dissafyt System Architecture

## Architecture Model

The platform uses a **monolithic Django architecture** to minimize operational complexity during early stages.

All components run inside a single backend application.

Future scaling can introduce service separation if required.

---

## Core Components

### 1 Marketing Website

Purpose:
Lead generation and customer acquisition.

Pages include:

/  
/pricing  
/templates  
/contact  
/signup  

---

### 2 Client Dashboard

Authenticated environment where users manage their website.

Features:

• onboarding wizard  
• business information management  
• media uploads  
• template selection  
• website publishing  

---

### 3 CMS Data Layer

Stores business information used to populate website templates.

Data includes:

• business details  
• services offered  
• gallery images  
• testimonials  
• contact information  

---

### 4 Website Rendering Engine

The Django engine dynamically renders websites using templates.

Rendering Logic:

Incoming Request  
↓  
Detect Domain or Subdomain  
↓  
Find Website Record  
↓  
Load Assigned Template  
↓  
Inject CMS Data  
↓  
Return Rendered Website  

---

## Example Request Flow

User visits:

plumber-1.dissafyt.co.za

System performs:

1 Identify host header  
2 Lookup website record  
3 Load template "plumber_template"  
4 Inject CMS data  
5 Render page  

---

## Hosting Model

All client websites are hosted within the Dissafyt Django platform.

There is no static generation.

Websites are dynamically rendered.
